Código Hamming.

En informática, el código de Hamming es un código detector y corrector de errores que lleva el nombre de su inventor, Richard Hamming. En los datos codificados en Hamming se pueden detectar errores en un bit y corregirlos, sin embargo no se distingue entre errores de dos bits y de un bit (para lo que se usa Hamming extendido).
Esto representa una mejora respecto a los códigos con bit de paridad, que pueden detectar errores en sólo un bit, pero no pueden corregirlo.

Ejemplo.
1.Consideremos la palabra de datos 0110101, agregaremos los bit de codigo en las posiciones 2n.
2.Procederemos a calcular los bit de codigo por sus posiciones.

P1 = D1+D2+D4+D5+D7 = 0+1+0+1+1 = 1
P2 = D1+D3+D4+D6+7 = 0+1+0+0+1 = 0
P2 = D2+D3+D4 = 1+1+0 = 0
P1 = D5+D6+D7 = 1+0+1 = 0

3. Tomaremos los 4 bits resultantes = 1000
4.Tener en cuenta que para saber el bit de codigo se mirará la cantidad de 1 que haya en la suma si es par el resultado será 0 y si no se agregará un 1.
5.Agregaremos estos bit resultantes en las posiciones 20,21,22,23.
6.El resultado sería 10001100101.

Detección de errores

Consideremos ahora que el bit de la derecha, por error, cambia de 1 a 0. La nueva palabra de datos será ahora 10001100100.
Para saber en donde se encuentra el error seguiremos los siguientes pasos.

1.Teniendo el código de Hamming completo proseguiremos a calcular nuevamente su XOR o verificaremos si es par o impar.
2.Esta vez se tendra en cuenta el bit P1, p2,..., Pn. Para saber el resultado.
3.Colocar el D7 = 0 ya que esté es el bit que viene con errores
4.Recuerda que los P1, p2,..., Pn. Se encuentra en las posiciones 2n.

P1+D1+D2+D4+D5+D7 = 1+0+1+0+1+0 = 1
P2+D1+D3+D4+D6+7 = 0+0+1+0+0+0 = 1
P3+D2+D3+D4 = 0+1+1+0 = 0
P4+D5+D6+D7 = 0+1+0+0 = 1

5.Tomado los bit de abajo hacia arriba tendriamos 1101 que en decimal equivale 11. 6.En la posición 11 se encuentra el bit errado, por lo tanto procedemos a corregirlo teniendo como resultado 10001100101.
